# 记忆情绪标注 - 设计文档

**日期**: 2026-04-22
**分支**: `feat/emotion-annotation`
**状态**: 实验性，数据采集阶段，召回逻辑暂未接入
**动机**: 为记忆增加情绪维度，使未来召回可基于情绪匹配（如低落时浮现温暖记忆，或同频共情）。

---

## 一、模型选择

采用 **Russell 环形情绪模型**（Circumplex Model of Affect），两个连续维度：

| 维度 | 含义 | 范围 | 正值 | 负值 |
|------|------|------|------|------|
| **valence** | 情绪效价（正面/负面） | -1.0 ~ 1.0 | 开心、感动、满足 | 难过、愤怒、焦虑 |
| **arousal** | 唤醒度（能量高/低） | -1.0 ~ 1.0 | 兴奋、激动、暴怒 | 平静、低落、疲惫 |

四象限：

```
        arousal +1.0
            │
   愤怒/焦虑 │ 兴奋/惊喜
  (excited-) │ (excited+)
 ────────────┼────────────  valence
    低落/无聊 │ 平静/满足
    (calm-)  │  (calm+)
            │
        arousal -1.0
```

**为什么不用离散分类**: 粒度太粗容易丢信息，太细模型判断不稳定。两个浮点数是连续值，标注稳定性高，且存储成本几乎为零。

## 二、改动位置

### 2.1 数据库迁移 — `aion-chat/database.py`

memories 表新增两列（向后兼容，ALTER TABLE 自动迁移）：

```python
("valence", "REAL DEFAULT 0.0"),
("arousal", "REAL DEFAULT 0.0"),
```

### 2.2 总结 prompt — `aion-chat/memory.py` (`_do_digest`)

在现有 JSON 输出的第 4 项（unresolved）后追加第 5、6 项：

```
5. "valence": (-1.0 ~ 1.0) 情绪效价。
6. "arousal": (-1.0 ~ 1.0) 情绪唤醒度。
   示例：惊喜收到礼物→valence:0.8,arousal:0.7；安静地回忆往事→valence:0.3,arousal:-0.5；...
```

解析时 clamp 到 [-1, 1]，写入 DB 并通过 WebSocket 广播。

**注意**: 总结调用走的是 `simple_ai_call`（核心模型），不依赖 sentinel，与上游兼容。

### 2.3 补标脚本 — `aion-chat/tools/backfill_emotion.py`

为已有记忆批量补充 valence/arousal。

关键设计：
- **输入是原始对话，不是摘要**。通过 `source_start_ts` / `source_end_ts` 回溯原始消息，摘要已经丢失了情绪信息。
- **模型 fallback**: 优先使用 sentinel（DashScope），`import` 失败时自动降级到 `simple_ai_call` + `DEFAULT_MODEL`，兼容上游无 sentinel 环境。

运行方式：
```bash
cd aion-chat
python tools/backfill_emotion.py
```

## 四、已知问题与局限

### 4.1 反讽/自嘲偏差

用户通过黑色幽默表达负面情绪时（如"笑死"、"搞笑了"），模型倾向标注为正面。表面语义与实际情绪反转，Russell 模型假设情绪可从文本表面读出，但讽刺型表达打破了这个假设。

**影响**: 特定用户的负面情绪被系统性低估。

**可能的缓解**:
- prompt 中增加反讽识别指引（增加标注复杂度，效果待验证）
- 结合上下文窗口而非单句标注（成本更高）

### 4.2 沉默缺失

不回复、不发起对话的沉默本身是强烈的情绪信号（calm- 极端），但在对话数据中完全缺失。模型只能标注"说了什么"，标不了"没说什么"。

**可能的缓解**:
- 结合消息间隔时间推断（如 >24h 未互动可标记为隐含低 arousal）
- 需要非文本信号（交互频率、在线时长等）

### 4.3 样本偏正面

初始 26 条记忆中绝大多数落在 excited+ 象限。可能原因：
- 用户确实与 bot 互动时情绪偏积极（真实分布）
- 小模型对中文对话有正面偏置（模型问题）
- 反讽表达被误标（4.1 的影响）

**需要更多用户数据验证**，尤其是情绪表达风格不同的用户。

### 4.4 摘要 vs 原文标注质量差异

实测对比：
- 摘要标注：API 挫败记忆被标为 v=-0.6, a=-0.4（强负面）
- 原文标注：同一条被标为 v=+0.2, a=+0.4（轻正面）

原因：摘要浓缩了负面事实描述，丢失了用户在解决过程中的积极情绪。**补标必须用原文**，新增记忆的标注已在 `_do_digest` 中与原始对话文本一起输入，无此问题。
